## Architectural Context for the Receiving Agent

### Two-Plane Design

- **OpenClaw** (Node.js, `npm install -g openclaw`) = control plane. Channel ingress, trust, routing.
- **OsMEN-OC** (this repo, Python 3.13) = execution plane. Pipelines, tools, memory, containers.
- Data flow: User → OpenClaw → WebSocket `ws://127.0.0.1:18789` → OsMEN-OC executes → results back.

### Key Directories

| Directory | Purpose |
|-----------|---------|
| `core/` | Installable Python package (config, models, bus, memory, DB, gateway, vision, voice, tasks) |
| `agents/` | YAML manifests — data, not code. MCP auto-registration reads these. |
| `config/` | YAML config truth store. `compute-routing.yaml` is the GPU/NPU routing policy. |
| `quadlets/` | Rootless Podman systemd unit files (NEVER Docker) |
| `tests/` | pytest + pytest-anyio. One test per module minimum. |
| `temp_1st_install/` | Install plan, log, reconciliation ledger |

### Critical Rules

- **Podman only** — no Docker, no docker-compose, no Dockerfile
- **No .env files** — secrets are SOPS-encrypted YAML under `~/.config/osmen/secrets/`
- **No requirements.txt** — use `pyproject.toml`
- **Python 3.13** — type hints, asyncio, pathlib, loguru, httpx, pydantic
- **Tests**: pytest + pytest-anyio, marker `@pytest.mark.anyio`
- **Approval gate**: every tool invocation goes through `core/approval/gate.py`
- **Event bus**: Redis Streams with typed `EventEnvelope` dataclass — never raw dicts
