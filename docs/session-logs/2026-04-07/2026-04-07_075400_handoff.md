## Decisions Made

1. **Python 3.13 explicitly**: System `python3` → 3.14.3, project requires 3.13. All venv commands use `python3.13` explicitly.
   - *Rationale*: pyproject.toml specifies `requires-python = ">=3.13,<3.14"`
   - *Impact*: Makefile and bootstrap.sh both patched

2. **Gateway port 8080**: Changed from 8000 to avoid conflict with ChromaDB container publishing on 8000.
   - *Rationale*: Both can't bind 127.0.0.1:8000
   - *Impact*: Makefile `make dev` target updated

3. **pkexec over sudo**: All privilege escalation uses pkexec (PolicyKit GUI dialog).
   - *Rationale*: Ubuntu 26.04 install has sudo-rs configured but pkexec is the working GUI escalation
   - *Impact*: bootstrap.sh patched (4 occurrences), staged

4. **Git identity**: `<your-github-username>` / `<your-github-username>@users.noreply.github.com`
   - *Rationale*: User confirmed "same git identity"; no public email on GitHub profile
   - *Impact*: To be applied in Phase 0

5. **168-step granularity**: User explicitly requested "no fewer than 100 items" for the install checklist.
   - *Impact*: Agent and todo list both carry the full 168-step plan

6. **Dedicated agent**: Created `final_first_install.agent.md` combining install_agent.md domain knowledge + INSTALL_INSTRUCTIONS.md workflow rules.
